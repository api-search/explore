This is the beta version of the APIs.io search engine to understand what is next. We are actively profiling new APIs and experimenting with different ways of organizing the APIs.json indexes and search--this site represents the latest of these iterations.

This round we have broken up my Artisanal repository that I was using to store all of the APIs.json for the API I had been profiling, but instead of publishing to a single search, I now have things broken into the network search and the individual search nodes representing a single topic.

Network

This website reflects what we would call a network APIs.io node, providing a single APIs.json that utilizes the new network property for APIs.json to list many different search nodes. There is only a single APIs.json in this website, which provides the search index for all of the others.

Search

APIs.io search nodes are individual topics with many APIs profiled, providing a single cluster of APIs that represent a specific industry or type of APIs. There are just a handful of nodes identified, pulling some of the most well known APIs we have profiled, with others coming soon.

The goal of breaking things into more modular chunks is to help reduce the processing power it takes to work with each node, but also reduce the cognitive load for anyone wanting to fork and put to work. However, we still want to evolve a search engine that will not just discovery new APIs.json and make them available for discovery, but also make it easier to manage and work with thousands of different types of APIs.
